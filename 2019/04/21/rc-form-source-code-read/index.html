rc-form-source-code-read

rc-form源码阅读

Posted by codinglobster on 2019-04-21

前言

state is a precious resource in react component, sometimes we don’t need to stick everything into it. if there’s a lot of value don’t need to react the action, just store it in the HOC at the very begining.

rc-form是阿里antd组件的基础，很好的分离了Form组件与业务组件的耦合

Form.create([options])

第一步，作为HOC,需要将已有组件进行包裹，传入对应的options,获取相应的设置,并返回一个接受组件的函数

Option Description Type Default
option.validateMessages Preseted messages of async-validator Object {}
option.onFieldsChange Called when field changed, you can dispatch fields to redux store. (props, changed, all): void NOOP
option.onValuesChange Called when value changed. (props, changed, all): void NOOP
option.mapProps Get new props transfered to WrappedComponent. (props): Object props => props
option.mapPropsToFields Convert value from props to fields. Used for read fields from redux store. (props): Object NOOP
option.fieldNameProp Where to store the name argument of getFieldProps. String -
option.fieldMetaProp Where to store the meta data of getFieldProps. String -
option.fieldDataProp Where to store the field data String -
option.withRef(deprecated) Maintain an ref for wrapped component instance, use refs.wrappedComponent to access. boolean false

getInitialState

由于Form HOC使用的旧的创建方式，所以state的初始化由getInitialState生命周期来完成，但是这个周期只是定义一些内部变量，state本身并不参与到组件的更新。唯一的state变量submitting已经废弃。

fieldStore用来保存所有form组件的field和metaField（原始信息和动态信息）
cachedBind用来保存所有namespace的回调
clearedFieldMetaCache当一个组件被卸载的时候，缓存它的field和metaField（原始信息和动态信息）
renderFields用来保存当前form组件是否已经渲染
domFields判断当前这个组件是否已经卸载组件，即是否拥有实例。

componentDidMounted & componentDidUpdate

在组件加载完成之后会进行一次清洗操作，删除无用的fields

即这个组件之前已经卸载了或者没有渲染，而且配置里也没有preserve的话，则清空他们在fieldStore里的记录。同时将当前的renderFields清空，因为下次renderField的修改是先于清空的。

getFieldDecorator & getFieldProps

使用这两个方法向fieldStore注册当前组件的fieldMeta,将验证的钩子挂载到组件的props,
并缓存一些组件的信息用来验证组件的状态。
同时通过改写ref的方法，来保存组件的实例，以及在组件卸载的时候将组件的field和metaField（原始信息和动态信息）保存到clearedFieldMetaCache。

这个时候组件的初始化就已经完成。

validate & store Field

组件在getFieldDecorator注册好了什么时候触发验证方法。比如onClick, onBlur。

onClickCommon是用来确认当前组件field,同时要触发原始组件的相应的事件。

在拿到组件的最新状态后，onCollectValidate会调用validateFieldsInternal进行规则验证。

第一步将需要验证的fields所有信息进行缓存，并通过
setFields将当前的组件状态更新到视图层。上面说到，HOC的刷新不是通过state完成的，而是通过每次setFields来forceUpdate().

通过第三方的AsyncValidator，结合从fieldMeta中拿到的验证设置，从而判断当前组件是否通过验证。并将更新后的每个组件状态通过setFields再更新一次。一次由用户操作引起的验证就已经完成。
